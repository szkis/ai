### 为什么不使用 Hexo 之类的静态站点

Hexo 的流程:

- 新建一个 xxx.md(无论是手动或命令行)
- 编辑相应的信息, 如发布日期, 分类, 标签, 内容.
- 本地预览
- 将 markdown 构建成 html 静态文件
- 部署到服务器
- push 源代码

它并不适合我:

- 疲与上面的步骤
- 依赖于开发环境，首先你要有 NodeJS 以及 Hexo
- 修改麻烦，有时就想要改一个字，还得打开编辑器，修改，build，部署.

> 我的目标是：部署之后，不用再 care 源码，也不依赖于生产环境，登陆 Github 就操作。
